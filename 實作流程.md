## 目標

建立一條完整的 GitHub Actions CI/CD 管線：
1. 每次 `git push` 自動執行 **lint + pytest**

## Step 2：重構 Dockerfile（移除 build-time GPU）（15 min）

### 為什麼要重構？

Lab 10 的 `Dockerfile.single` 在 `RUN` 階段執行 `yolo export format=engine`，需要 `--runtime nvidia`。
GitHub 託管的 x86 runner 無法提供 CUDA 裝置，因此必須將引擎編譯移至容器啟動時（`entrypoint.sh`）。

### 2.1 已完成的修改

| 原始（Lab 10） | 修改後（Lab 12） |
|---|---|
| `RUN ... yolo export format=engine` | ❌ 移除 |
| 無 entrypoint | `ENTRYPOINT ["/app/entrypoint.sh"]` |

`entrypoint.sh` 邏輯：
1. 檢查 `best.pt` 是否存在
2. 若 `best.engine` 不存在或比 `best.pt` 舊 → 編譯（5–8 min）
3. 否則直接重用快取
4. `exec "$@"` 確保 SIGTERM 直達 Python（優雅關機）

### 2.2 提交並推送

```bash
git add entrypoint.sh Dockerfile.ci validate_step2.sh
git commit -m "Defer TRT engine compile to container entrypoint"
git push
```

### 驗證 Step 2

```bash
chmod +x validate_step2.sh
./validate_step2.sh
```

預期輸出：
- ✓ entrypoint.sh present + executable
- ✓ shebang OK
- ✓ Dockerfile.ci wires ENTRYPOINT
- ✓ no build-time engine compile in Dockerfile.ci
- **Step 2 PASS**


### 3.2 等待 Build 完成

Build job 流程：
1. `docker/setup-qemu-action@v4` — 啟用 QEMU 模擬 arm64
2. `docker/setup-buildx-action@v4` — 安裝 BuildKit
3. `docker/login-action@v4` — 用 `GITHUB_TOKEN` 登入 ghcr.io
4. `docker/metadata-action@v6` — 產生 `sha-<commit>` 與 `latest` 標籤
5. `docker/build-push-action@v7` — 跨架構編譯並推送

等待期間可做：
- 在 Jetson 預先拉取 base image：`docker pull dustynv/pytorch:2.7-r36.4.0`
- 提前閱讀 Step 4

## Step 4：在 Jetson 上拉取並執行映像（15 min）

> 以下步驟在 Jetson 上執行

### 4.1 事前檢查

```bash
# 磁碟空間 ≥15 GB
df -h /var/lib/docker | tail -1

# nvidia runtime 已設定
docker info 2>/dev/null | grep -i nvidia

# Mosquitto 運行中
systemctl status mosquitto --no-pager | head -5
```

### 4.2 登入 GHCR

```bash
# 安裝 gh（若尚未安裝）
sudo apt update && sudo apt install -y gh
gh auth login

# 取得 token 並登入 docker
gh auth token | docker login ghcr.io -u <your-github-user> --password-stdin
```

### 4.3 拉取映像

```bash
# 從 Jetson 上的 lab12 clone 取得 SHA
cd ~/lab12 && git pull
SHA=$(git rev-parse --short HEAD)
GHCR_USER=<your-github-user>
IMAGE="ghcr.io/${GHCR_USER}/edgeai-lab12:sha-${SHA}"

# 拉取（首次 1–4 min）
docker pull "$IMAGE"
```

### 4.4 建立 volume 並執行

```bash
docker volume create lab12-models

TEST_VIDEO=~/lab10/test_video.mp4

docker run --rm -it \
    --runtime nvidia \
    --network host \
    -v lab12-models:/opt/models \
    -v "$TEST_VIDEO":/opt/data/test_video.mp4:ro \
    "$IMAGE"
```

**首次啟動**（5–8 min）：
```
[entrypoint] Compiling TensorRT engine (this takes 5-8 min on first boot)...
[entrypoint] Engine compiled: 24M
[inference] Loading /opt/models/best.engine...
[inference] Running inference on /opt/data/test_video.mp4...
[inference] 100 frames, 38.5 FPS, last frame: 2 detections
```

**再次啟動**（~5 s）：
```
[entrypoint] Reusing cached engine: 24M
[inference] Running inference on /opt/data/test_video.mp4...
```

### 驗證 Step 4

```bash
chmod +x validate_step4.sh
GHCR_USER=<user> SHA=<sha> TEST_VIDEO=~/lab10/test_video.mp4 ./validate_step4.sh
```

預期輸出：
- ✓ test video found
- ✓ image pulled
- ✓ arch = arm64
- ✓ entrypoint compiled or reused engine
- ✓ inference_node.py started and is processing the test video
- **Step 4 PASS**

## Step 5（Stretch）：自架 Jetson Runner（10 min）

> 選做，完成後 HW6 可直接使用。

### 5.1 下載並註冊 runner

```bash
mkdir -p ~/actions-runner && cd ~/actions-runner
RUNNER_VER=2.319.1
curl -O -L "https://github.com/actions/runner/releases/download/v${RUNNER_VER}/actions-runner-linux-arm64-${RUNNER_VER}.tar.gz"
tar xzf actions-runner-linux-arm64-${RUNNER_VER}.tar.gz

# 從 GitHub UI 取得 registration token：
# Repo → Settings → Actions → Runners → New self-hosted runner → Linux ARM64
TOKEN=<paste-token-here>

./config.sh \
    --url "https://github.com/$(gh repo view --json nameWithOwner --jq .nameWithOwner)" \
    --token "$TOKEN" \
    --labels "self-hosted,linux,arm64,jetson,orin-nano" \
    --name "jetson-lab12" \
    --unattended
```

### 5.2 以 systemd service 運行

```bash
sudo ./svc.sh install
sudo ./svc.sh start
sudo ./svc.sh status  # 應顯示 active (running)
```

### 5.3 新增 jetson-smoke job

在 `.github/workflows/ci.yml` 最後加入：

```yaml
  jetson-smoke:
    name: Smoke test on Jetson
    runs-on: [self-hosted, linux, arm64, jetson]
    needs: build
    steps:
      - uses: actions/checkout@v5
      - name: Pull and verify image on this Jetson
        run: |
          SHORT_SHA=$(echo "${{ github.sha }}" | cut -c1-7)
          IMAGE="ghcr.io/${{ github.repository }}:sha-${SHORT_SHA}"
          echo "${{ secrets.GITHUB_TOKEN }}" | docker login ghcr.io -u ${{ github.actor }} --password-stdin
          docker pull "$IMAGE"
          docker volume create lab12-models
          docker run --rm --runtime nvidia -v lab12-models:/opt/models \
            "$IMAGE" python3 -c "
from ultralytics import YOLO
m = YOLO('/opt/models/best.engine', task='detect')
print('PASS: engine loads on Jetson hardware')
"
```

### 5.4 清理（TA 驗收後）

```bash
cd ~/actions-runner
sudo ./svc.sh stop
sudo ./svc.sh uninstall
./config.sh remove --token <removal-token>
cd ~ && rm -rf ~/actions-runner
```

### 驗證 Step 5

```bash
chmod +x validate_step5.sh
REPO=<user>/edgeai-lab12 ./validate_step5.sh
```

## 常見問題排解

### gh 認證問題

```bash
# token 過期
gh auth refresh

# Jetson 與筆電各自需要獨立認證
gh auth login  # 在 Jetson 上也要執行
```

### GHCR 無法 pull（denied）

```bash
# 將 package 設為 public
# Repo Settings → Packages → edgeai-lab12 → Change visibility → Public
```

### build job 無 arm64 manifest

檢查 `.github/workflows/ci.yml` 中的 `build` job 是否包含：
```yaml
platforms: linux/arm64
```

### 引擎編譯掛住（>15 min）

```bash
# 釋放 GPU 記憶體
sudo systemctl stop nvargus-daemon
nvidia-smi  # 確認無殘留程序
# 重新執行 docker run
```

### 每次重啟都重新編譯

確認 `docker run` 有加上 volume mount：
```bash
-v lab12-models:/opt/models
```
